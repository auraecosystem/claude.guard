- The sandbox image now bakes `uv` (the Python launcher) alongside the existing `python3`/`pip`/`detect-secrets`. `session-setup.sh` gates every Python provisioning step (`uv sync` for the project `.venv`, `uv tool install` for ruff/zizmor/pre-commit) behind `command -v uv`, so on an image without `uv` a guarded repo silently got no Python toolchain. Baking it removes the per-session network bootstrap and lets provisioning work under restricted outgoing access.
