- MCP server approve/reject decisions now persist across ephemeral sessions. The first time you approve (or reject) a project's `.mcp.json` servers, the choice is remembered on a durable, cross-project volume and re-applied at the next session start — so a wiped ephemeral `$HOME` no longer re-prompts for every server on every launch. A changed server definition still re-prompts (the remembered decision is fingerprint-gated), and a restored approval/rejection is announced in the session-start notice rather than applied silently.
