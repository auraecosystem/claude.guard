- The publish-time image-verification CI gate (`.github/scripts/publish-image-verify-public.sh`) now enforces the same trust anchor a default consumer does: it pins the commit (`--certificate-github-workflow-sha`) and reaches the Rekor-dropping TSA fallback only under the explicit `CLAUDE_GUARD_COSIGN_ALLOW_TSA_FALLBACK=1` opt-in. Previously the gate omitted the commit pin and ran the TSA fallback unconditionally, so a release could go green while default consumers rejected the published image.
