- Image GC now also reclaims superseded local sandbox builds — the dangling (`<none>`) images a `docker compose build` orphans when it re-tags `secure-claude-sandbox:local` onto a freshly built image. Previously these accumulated unbounded on hosts that build locally (the image reaper only touched prebuilt `git-<sha>` sets, and skipped entirely on a dirty/non-candidate launch). They're identified by the `claude-guard.git-commit` build label, so unrelated dangling images are never touched, and an image still backing a container is left in place. Runs on each launch and via `claude-guard gc`; opt out with `CLAUDE_NO_IMAGE_GC=1`.
