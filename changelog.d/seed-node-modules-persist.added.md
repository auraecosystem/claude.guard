- Seed-mode launches now persist `node_modules` in a per-workspace cache volume (`claude-guard-node-modules-<hash>`), so a cold relaunch reuses the populated tree (hitting the stat-only "already current" fast path) instead of rebuilding it from empty — collapsing the dominant cold-launch install leg. The volume is written only by the in-container hardener (never the host, so it is cross-platform-safe), kept read-only to the agent, and reclaimed by the existing workspace-deletion GC. Bind mode is unaffected.
