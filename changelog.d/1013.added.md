- `claude-guard gh-app setup`: creates and installs the GitHub App in one walkthrough, instead of running `create` then `install` separately.
- `claude-guard gh-app verify`: mints a token and confirms it actually authorizes against the installation — an end-to-end health check of the whole GitHub-access chain that `status` can't give.
- GitHub App tokens are cached briefly (`CLAUDE_GH_TOKEN_CACHE_TTL`, default 90s, `0` disables) on the memory-backed runtime dir, so rapid relaunches skip the GitHub round-trip. A reused token always retains nearly its full ~1h life, so a working session is never handed a near-expiry token.
- `git config --local claude-guard.token-repos <list|all>`: pin a repo's GitHub App token scope once instead of exporting `CLAUDE_GH_TOKEN_REPOS` each session (read repo-local only, so a global setting can't silently widen scope).
- `CLAUDE_GH_APP_KEYCHAIN=file`: force the App private key to the on-disk `0600` backend for headless/SSH hosts whose OS keyring is locked or absent.
