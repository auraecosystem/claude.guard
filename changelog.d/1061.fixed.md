- The read-only guardrail protection-layer generator now refuses to install a truncated or empty Compose override over a working one: a generator whose write fails or produces no output is rejected with a clear warning and the existing override is kept, so a half-written file can never silently drop the read-only binds that protect `.claude`/`.devcontainer`/`node_modules`/`CLAUDE.md`/`AGENTS.md`.
- The Docker daemon restart performed when registering the sandbox runtime now confirms the daemon is actually answering again afterwards (a bounded `docker info` poll) instead of trusting the restart command's exit code, so a restart that returns before the daemon is back is reported as a failure rather than a false success.
- Orphaned-session and pre-warm-spare reapers now emit a loud, attributable warning naming the leftover project and volume id when a teardown fails, instead of swallowing the failure — a leaked session's volumes/network are now visible in the launch log.
- Secrets written through the privileged atomic-write helper now have their temp file tightened to the destination's mode before the content is written when that mode is stricter than owner-only, so the bytes never momentarily exist at a wider permission than intended.
