- The monitor's ntfy push now reads an optional `token=` line from `~/.config/claude-monitor/ntfy.conf` and sends it as an `Authorization: Bearer` header, so alerts can target an access-controlled topic (a self-hosted ntfy via `url=`, or a reserved `ntfy.sh` topic) instead of only the public namespace. The token is treated as a credential — sent only in the header, never placed in the URL/body or logged.
