- The dev-session deny list no longer blocks the privilege helpers `sudo`, `su`, `capsh`, and `setcap`. Inside the sandbox these can't escape the gVisor/cap-drop floor, so the rules added no real protection while blocking everyday privileged setup (`sudo apt-get install …`). The specific destructive/firewall rules (`rm -rf`, `dd`, `iptables`, `ip route`, …) still apply to their bare form; a `sudo`-prefixed invocation is now an accepted speed-bump gap, exactly like the already-accepted path-prefixed (`/bin/dd`) form.
