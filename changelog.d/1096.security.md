- Egress firewall: GitHub IP ranges broader than `/16` (was `/8`) are now ignored rather than admitted, capping a spoofed `api.github.com/meta` response at 64K addresses instead of 16M. An over-broad-but-well-formed range is skipped with a warning (GitHub stays reachable via DNS) rather than aborting the firewall, while a structurally corrupt entry still fails loud.
- Per-project allowlist: punycode/non-ASCII (`xn--`) domains from a workspace's settings are now rejected by default as lookalike-domain risks. Set `CLAUDE_GUARD_ALLOW_PROJECT_IDN=1` to admit genuine IDN hosts.
